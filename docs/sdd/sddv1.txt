2.1 Arsitektur Sistem & Struktur Deployment
Aplikasi dibangun menggunakan Component-Based Architecture memanfaatkan framework React/Next.js
[cite: 13]. Seluruh halaman dikompilasi ke dalam aset statis melalui metode Static Site Generation (SSG)
dan dideploy di infrastruktur global Vercel [cite: 13]. Pendekatan ini dipilih guna mengeliminasi latensi
pemrosesan server-side pada runtime, menghasilkan skor *Core Web Vitals* yang optimal untuk menunjang
indeksasi SEO Lokal [cite: 13].

2.3 Desain Komponen (UI Logic & Analytics Integration)
`Layout.js`: Komponen pembungkus tingkat teratas (Root Layout). Bertanggung jawab menginjeksikan
script global Google Tag Manager / Meta Pixel secara asinkron tanpa memblokir proses rendering
komponen UI utama [cite: 16].
`HeroSection.js`: Komponen visual pertama di bagian atas halaman [cite: 17]. Menyediakan tombol aksi
utama yang memicu fungsi pengiriman event analitik secara spesifik (`click_wa_hero`) sesaat sebelum
mengarahkan pengguna ke tautan WhatsApp [cite: 17].
`ServiceCard.js`: Komponen kartu modular yang menerima properti (*props*) dinamis yang dipetakan
langsung dari JSON `layanan` [cite: 18]. Mengontrol penataan visual deskripsi, label harga, serta
menyusun format URL WhatsApp (menggunakan parameter `encodeURIComponent` untuk properti
`template_wa`) [cite: 14, 18].
`ServiceCatalog.js`: Komponen grid yang menggantikan fungsi `ProductCatalog.js` [cite: 18]. Komponen
ini mengimplementasikan logika state internal React (`useState`) untuk menyaring baris data berdasarkan
nilai `kategori` (perawatan, perbaikan, instalasi) tanpa memerlukan pemuatan ulang halaman (*zero-reload
filtering*).
`ContactButton.js`: Komponen tombol mengambang (*floating action button*) yang menetap di pojok
kanan bawah layar [cite: 19]. Tombol ini dibekali pelacak analitik mandiri yang merekam pemicu event
(`click_wa_floating`) guna membedakan konversi organik dari katalog dan konversi langsung [cite: 19].
